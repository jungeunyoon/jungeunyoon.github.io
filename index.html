Awards: "Best Overall App" and "Most Original Idea" @ 2014 UT Apps Demo Day

- - - - - - - - - -

Premise: An application that suggests the best outfit with respect to user ratings and the weather.

Team: Six members consisting of three CS an three Journalism students.

Tools Used: Xcode 5, Objective-C, TestFlight, CocoaPods, Parse, OpenWeatherMap API, Trello, Github, Git, Photoshop CS5, Illustrator, Final Cut Pro

Role:

  • We had a little less than two months to pick up iOS development without any instruction
  • Developed the outfit matching algorithm which suggests the best outfit (out of over 5000 possibilites) with respect to user ratings and the weather
  • Combined our separate code into one app
  • UI/UX design, Logo design and all the graphics
  • Presented the demo on demo day
  • Taught non-programmer teammates how to code

Value:
1. Communication & Problem Solving - This fast-paced class makes you fully realize that people naturally think differently. It forces you to learn how to be a better communicator and pushes you be a team player. It gave us a sense of the real world work environment.

2. Quick Learning - It taught us to learn and adapt quickly. We had a little less than two months to pick up iOS mobile development. It was either pick it up very quickly by investing time outside of classes or not have an app ready by demo day.

  • WEB APPLICATION


Premise: Dota 2 players commonly trade and sell items that they have acquired. This web application will provide a gateway to a database that lists the price data and trend of these items, among other things

Team: 5 members, "ItTakesFiveToDjango"

Tools Used: Rackspace, Apiary, Python, Javascript, PostgreSQL, CSS, HTML, Bootstrap, Django, jQuery, Github, Git, Slack

Role:

  • Led the team for Phase I (of three)
  • The overall task organizer for subsequent weeks
  • Quickly learned Twitter Bootstrap and created a custom responsive front end within two days
  • Wrote 85% of the 6000+ word technical report
  • created the data models and the UML diagrams

Value:
1. Collaboration - My team members and I were constantly communicating through Slack during those three weeks. One person would point out a minor bug and we, as a group, would get them fixed right away. It showed me even further how beneficial it is to work in a collaborative and supportive setting.

2. Adaptivity - We started off using a Bootstrap template because our professor encouraged all the groups to do so. He didn't see the value in any of us creating custom layouts and frankly, he didn't think that it was possible for us to come up with them in such a short amount of time and in the end, no other team used anything other than themes found online. However, I wanted to use the time that was given to actually learn so I spent three days learning Twitter Bootstrap (D-1), designing the website and making the graphics (D-2), and implementing the design (D-3). It was a great learning experience on how adaptive you have to be in the real world since job requirements and software is constantly changing.
